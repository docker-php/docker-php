Docker PHP

Docker PHP (for lack of a better name) is a Docker client written in PHP. This library is still a work in progress. Not much is supported yet, but the goal is to reach 100% API support.

The test suite currently passes against the Docker Remote API v1.9.

Installation

The recommended way to install Docker PHP is of course to use Composer:

{
    "require": {
        "stage1/docker-php": "@dev"
    }
}

Note: there is no stable version of Docker PHP yet.

Usage

See the documentation.

Unit Tests

Setup the test suite using Composer:

$ composer install --dev

Run it using PHPUnit:

$ bin/phpunit

Contributing

Here are a few rules to follow in order to ease code reviews, and discussions before maintainers accept and merge your work.

  • You MUST follow the PSR-1 and PSR-2.
  • You MUST run the test suite.
  • You MUST write (or update) unit tests.
  • You SHOULD write documentation.

Please, write commit messages that make sense, and rebase your branch before submitting your Pull Request.

One may ask you to squash your commits too. This is used to "clean" your Pull Request before merging it (we don't want commits such as fix tests, fix 2, fix 3, etc.).

Also, when creating your Pull Request on GitHub, you MUST write a description which gives the context and/or explains why you are creating it.

Thank you!
